# 示例说明

本目录使用"个人书架"作为示范项目，目的是展示**每个模块应该写到什么程度**。

## 如何使用这个示例

**参考的是格式和严谨度，不是具体内容。**

- ✅ 学习：字段怎么列、流程怎么分步骤写、限制怎么给具体数字
- ❌ 不要模仿：具体的产品功能、技术选型、页面布局

无论你做什么产品，PRD 的格式标准是一样的——具体、无歧义、可执行。

## 示例项目

| 模块 | 文件 | 核心示范点 |
|------|------|-----------|
| 产品意图 | 01-product-intent.md | 定位一句话说清、"不做什么"写成清单 |
| 技术约束 | 02-tech-constraints.md | 每个技术有版本号 + 一句理由 |
| 数据与接口 | 03-data-interfaces.md | 每个字段有类型 + 说明、API 有完整定义 |
| 交互与设计 | 04-interaction-design.md | 页面有 ASCII 布局图、操作分步骤写 |
| 验证与边界 | 05-verification.md | 成功标准是可执行的、每个依赖有兜底方案 |
