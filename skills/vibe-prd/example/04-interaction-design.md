## 3. 核心操作流程

### 流程 A：添加书籍

1. 用户点击 [+ 添加书籍] → 弹出添加表单（Modal）
2. 用户输入书名（必填）、作者（选填）→ 点击 [搜索]
3. 前端调用 `GET /api/search?q=xxx` → 展示搜索结果列表
4. 用户点击某条结果 → 表单自动填充（书名、作者、封面、ISBN、简介）
5. 用户选择阅读状态（默认"想读"）→ 点击 [保存]
6. 前端调用 `POST /api/books` → 收到 201 响应
7. Modal 关闭，书籍网格刷新，新书出现在最前面

### 流程 B：切换阅读状态

1. 用户在书籍卡片上点击状态标签（想读/在读/读完）→ 状态切换
2. 前端调用 `PUT /api/books/:id { status: "new_status" }` → 收到 200
3. 卡片上的状态标签更新
4. 如果当前筛选条件不包含新状态，卡片从当前视图消失

### 流程 C：AI 推荐

1. 用户点击 [AI 推荐下一本书] → 跳转到 /recommend
2. 前端调用 `POST /api/recommend` → 开始接收 SSE 事件
3. 界面依次显示：分析状态 → 候选书籍信息 → 推荐理由（逐字流式）
4. 用户点击 [+ 加入书架] → 调用 `POST /api/books`，书籍状态为"想读"
5. 用户点击 [换一本] → 重新调用推荐接口（排除已推荐过的书）

## 4. 设计规范

### 颜色

| 语义 | 色值 | 用途 |
|------|------|------|
| Primary | #5C4033 | 主要强调（深棕色，书卷气） |
| Secondary | #8B7355 | 次要强调（暖棕色） |
| Surface | #FAF6F0 | 页面背景（米白色） |
| On Surface | #2C2420 | 主要文字 |
| Want | #D4A574 | "想读"状态标签 |
| Reading | #6B8E6B | "在读"状态标签 |
| Done | #5B7B9E | "读完"状态标签 |
| Border | #E8E0D8 | 边框/分割线 |

### 字体

| 用途 | 字体 | 说明 |
|------|------|------|
| 书名/标题 | Noto Serif (serif) | 衬线体，营造书卷气 |
| 正文/UI | Manrope (sans-serif) | 无衬线，现代可读 |

### 间距与圆角

| Token | 值 | 用途 |
|-------|------|------|
| 基础单位 | 4px | 基准 |
| 容器内边距 | 16px | 卡片内部 |
| 元素间距 | 12px | 元素之间 |
| 默认圆角 | 12px | 卡片/容器 |
| 按钮圆角 | 8px | 按钮 |
| 标签圆角 | 999px | 状态标签（胶囊形） |

## 5. 状态定义

| 组件 | 空状态 | 加载状态 | 错误状态 |
|------|--------|---------|---------|
| 书籍网格 | 显示空书架插画 + "书架空空的" + [添加第一本书] 按钮 | 6 个灰色骨架卡片 (shimmer) | 红色提示 + [重试] 按钮 |
| 搜索结果 | "没找到相关书籍" + 建议手动添加 | spinner 旋转图标 | "搜索服务暂不可用" + 手动输入选项 |
| AI 推荐 | 未触发时显示 "点击开始推荐" | 流式状态消息 + spinner | "推荐服务暂不可用" + [重试] 按钮 |
| 书籍详情 | - | 骨架屏 | "书籍不存在" + [返回首页] |
| 读书笔记 | "点击添加笔记..." 灰色占位文字 | - | 保存失败 toast 提示 |

## 6. 响应式

| 断点 | 布局 |
|------|------|
| 桌面 (≥768px) | 书籍网格 4-6 列，添加书籍为 Modal 弹窗 |
| 平板 (600-767px) | 书籍网格 3 列 |
| 手机 (<600px) | 书籍网格 2 列，添加书籍改为全屏页面，筛选栏可横向滚动 |
