# 05 · 验证与边界 — 格式示范

> ⚠️ **这是格式示范，不是内容模板。** 参考的是"成功标准怎么写成可执行步骤、兜底策略怎么按依赖列"，不是"应该测什么功能"。
>
> 对应模板：`templates/05-verification.md`

---

## 1. 成功标准

1. 运行 `npm run dev`，浏览器打开 `http://localhost:3000` → 看到空书架页面（空状态插画 + "书架空空的"）
2. 点击 [+ 添加书籍] → 弹出添加表单
3. 输入书名"百年孤独"，点击搜索 → 看到搜索结果列表
4. 点击第一条结果 → 表单自动填充书名、作者、封面
5. 选择状态"想读"，点击保存 → 书籍出现在书架网格中，显示封面和"想读"标签
6. 点击书籍卡片 → 进入详情页，显示完整信息
7. 点击状态标签切换为"在读" → 标签颜色从暖棕变为绿色
8. 在详情页输入读书笔记并保存 → 刷新后笔记内容还在
9. 回到首页，点击 [AI 推荐下一本书] → 看到流式推荐（状态消息 → 书籍信息 → 推荐理由逐字出现）
10. 在推荐结果中点击 [+ 加入书架] → 书籍出现在书架中，状态为"想读"
11. 刷新页面 → 所有书籍和笔记数据都在

## 2. 兜底策略

| 外部依赖 | 失败场景 | 处理方式 |
|----------|---------|---------|
| aihubmix (AI 推荐) | 超时 (>60s) | 显示"推荐超时，请稍后重试" + [重试] 按钮，不影响其他功能 |
| aihubmix (AI 推荐) | API Key 无效 | 显示"AI 服务未配置，请检查 API Key"，推荐按钮置灰 |
| aihubmix (AI 推荐) | 服务不可达 | 显示"推荐服务暂不可用"，推荐页面保持可浏览状态 |
| Tavily (搜索) | 超时 (>15s) | 搜索框降级为纯手动输入（用户直接输入书名保存） |
| Tavily (搜索) | API Key 未配置 | 搜索功能不可用，添加表单切换为手动输入模式，不显示搜索按钮 |
| Google Books API | 无结果 | 显示"未找到结果，请手动输入" |
| Google Books API | 服务不可达 | 搜索静默失败，降级为手动输入 |
| SQLite | 数据库文件损坏 | 启动时检测，提示"数据库异常" + 提供重建选项（会丢失数据） |
| SQLite | 磁盘空间不足 | 保存操作失败时 toast 提示"磁盘空间不足" |

## 3. 性能限制

| 限制项 | 值 | 说明 |
|--------|------|------|
| 单本书名最大长度 | 200 字符 | 超过截断 |
| 读书笔记最大长度 | 50,000 字符 | 约 3 万字，足够写完整读书笔记 |
| 书架最大书籍数 | 5,000 本 | 超过后性能会下降（前端网格渲染） |
| 搜索结果数量 | 最多 20 条 | Google Books API 单次返回上限 |
| AI 推荐候选数 | 最多 3 本 | 一次推荐返回 3 本候选 |
| 封面图片大小 | 最大 5MB | 上传限制（如果支持本地上传） |
| 搜索请求频率 | 10 次/分钟 | 防止频繁调用外部 API |

## 4. V1 不做什么

- 不做导出功能（不导出 CSV/PDF/笔记）
- 不做暗色模式
- 不做阅读统计/数据可视化（阅读量图表等）
- 不做批量导入（如从豆瓣导入）
- 不做阅读进度追踪（如"读到第 50 页"）
- 不做书籍封面本地上传（只用外部 URL）

## 5. 降级方案

| 核心功能 | 依赖不可用时 | 降级表现 |
|----------|------------|---------|
| AI 推荐 | aihubmix 不可用 | 推荐按钮置灰 + 显示"推荐暂不可用"，书架的其他功能完全正常 |
| 搜索添加 | Tavily + Google Books 都不可用 | 添加表单降级为纯手动输入（用户自己填写书名、作者、封面 URL） |
| 封面显示 | 封面 URL 失效/无法加载 | 显示默认占位封面（书本图标 + 书名首字母） |
| 标签筛选 | - | 标签功能无外部依赖，不存在降级问题 |
