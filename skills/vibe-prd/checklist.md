# Vibe Coding PRD — 自查清单

写完 PRD 后逐项检查。每个问题必须回答"是"才算合格。打勾 × 表示不合格，需要补充。

---

## 模块 1 — 产品意图

- [ ] 产品定位是否一句话能说清？
- [ ] 目标用户是否具体到"某类人"而非"所有人"？
- [ ] 核心价值是否可验证（用户能得到什么具体结果）？
- [ ] 品牌调性是否有具体关键词或参考？
- [ ] "不做什么"是否明确列出？

## 模块 2 — 技术约束

- [ ] 每个技术层是否指定了具体技术 + 版本？
- [ ] 每个选型是否有一句话理由？
- [ ] 部署方式是否明确（本地/云/容器）？
- [ ] 所有环境变量是否列出？
- [ ] "禁止用什么"是否明确列出？

## 模块 3 — 数据与接口

- [ ] 每个实体是否列出了所有字段（名称 + 类型 + 说明）？
- [ ] 实体间关系是否用文字或图描述清楚？
- [ ] ID 命名规则是否统一？
- [ ] 每个 API 是否定义了方法 + 路径 + 请求体 + 响应体？
- [ ] 流式接口的事件序列是否明确定义？（如适用）

## 模块 4 — 交互与设计

- [ ] 是否列出了所有页面及其路径？
- [ ] 核心操作流程是否分步骤写清楚了？
- [ ] 设计规范是否精确到值（颜色 HEX / 字体名 / 间距 px）？
- [ ] 每个关键组件的空状态和加载状态是否定义了？
- [ ] 桌面 / 移动端差异是否说明了？

## 模块 5 — 验证与边界

- [ ] 成功标准是否是一条可执行的端到端流程？
- [ ] 每个外部依赖是否有失败时的处理方案？
- [ ] 性能限制是否有具体数字？
- [ ] "不做什么"是否与模块 1 一致且更具体？
- [ ] 是否定义了降级方案（核心功能不可用时怎么办）？

## 全局检查

- [ ] 5 个模块之间是否一致？（模块 1 说"不做登录"，模块 4 里就没有登录页）
- [ ] 一个非技术人员读完是否能理解？
- [ ] AI 读完是否能直接开始写代码，不需要额外追问？
- [ ] 如果删除某个模块，AI 是否会缺少关键信息而无法执行？

---

## 常见不合格原因

| 症状 | 根因 | 修复方式 |
|------|------|---------|
| AI 问了很多"确认"问题 | PRD 有歧义 | 检查模块 1 和模块 2 是否足够具体 |
| AI 加了不需要的功能 | 没写"不做什么" | 补充模块 1 和模块 5 的排除清单 |
| AI 用了你不想要的技术栈 | 没写技术约束 | 补充模块 2 |
| AI 在不同文件里类型不一致 | 数据模型没定义清楚 | 补充模块 3 |
| 每个页面长得不一样 | 没写设计规范 | 补充模块 4 |
| 出错时白屏或崩溃 | 没写兜底策略 | 补充模块 5 |
